## 1. 项目定位

面向资金规模约10万～1000万元的个人投资者，构建一个以A股为主要交易市场、预期持有40～80个交易日、同时持有3～5只股票的中期波段决策系统。

系统不追求日内高频交易，而是将全球风险环境、A股市场状态、题材生命周期、资金与成交结构、公司质量和价格趋势组合起来，每日收盘后给出少而精、可解释、可执行的持仓建议。

本项目定位为“投资决策支持＋模拟组合”，不连接券商、不接收交易凭据、不自动或半自动下单。若未来需要真实交易能力，必须作为独立项目重新进行产品、安全与合规评审，不属于本方案交付范围。

> 本产品提供研究和决策辅助，不承诺收益，也不能替代用户自行判断或持牌投资顾问服务。

## 2. 已确定的产品边界

| 项目 | 当前定义 |
|---|---|
| 交易市场 | A股 |
| 目标用户 | 个人投资者 |
| 资金规模 | 10万～1000万元 |
| 正式持仓 | 3～5只，默认4只 |
| 备选池 | 2～3只，不计入正式持仓 |
| 预期持有 | 约3个月，即40～80个交易日 |
| 决策频率 | 周频选题材和调仓，日频监控风险 |
| 组合风格 | 中期趋势＋产业/事件催化＋精选个股 |
| 止盈偏好 | 浮盈超过门槛后，最多回吐最高浮盈的30% |
| 调仓限制 | 正常情况下每周最多替换1只 |

尚需区分两个概念：

1. “浮盈回吐30%”用于单只股票的移动止盈。
2. “账户最大回撤”用于组合生存控制。第一版暂按目标不超过15%、硬风控18%设计，最终需由产品负责人确认。

## 3. MVP要解决的问题

每个交易日收盘后，系统需要回答五个问题：

1. 当前A股处于什么市场状态，建议总仓位是多少？
2. 哪3～4个题材处于启动、扩散或健康趋势阶段？
3. 哪3～5只股票最值得持有，为什么？
4. 每只股票的目标仓位、买入触发、失效条件和止盈止损是多少？
5. 现有持仓应该继续持有、加仓、减仓还是退出？

MVP明确不做：

- 分钟级高频策略；
- 自动追涨停或打板；
- 使用无法验证的“主力资金”作为核心事实；
- 由大语言模型直接预测涨跌；
- 任何自动或半自动下单、券商连接与真实订单管理；
- 收益保证或个性化代客理财。

## 4. 用户端核心体验

首页只呈现一个精选组合，而不是几十只股票的排行榜。

### 4.1 每日组合卡片

每只股票显示：

- 股票与所属题材；
- 当前状态：观察、待买、持有、加仓、减仓、退出；
- 推荐目标仓位和当前仓位；
- 入选原因，限制在3条以内；
- 买入触发区间；
- 初始止损或逻辑失效条件；
- 当前最高浮盈和移动保护价；
- 预计持有区间和下次复核时间；
- 关键风险：财报、减持、解禁、监管、流动性等。

### 4.2 市场温度页

展示：

- 市场状态：强势、震荡偏强、震荡偏弱、下跌、极端风险；
- 建议总仓位；
- 全球风险、A股广度、流动性、风格和情绪五个分项；
- 当前主导风格：大盘/小盘、价值/成长、周期/防御；
- 状态发生变化的主要原因。

### 4.3 题材雷达

每个题材标注其生命周期：潜伏、启动、扩散、加速、拥挤、退潮。用户能看到题材强度、内部广度、成交占比、催化持续性和拥挤度，而不是只看到新闻热度。

### 4.4 决策日志

每次信号必须保存当时的数据快照、模型版本、推荐原因和后续结果，用于回看“当时为什么这么判断”，避免事后改口。

## 5. 数据方案

### 5.1 必需数据

1. A股日线行情：开高低收、成交量、成交额、停牌、涨跌停状态。
2. 复权和公司行为：分红、送转、拆并股、配股。
3. 股票基础信息：上市/退市日期、板块、ST状态、流通市值。
4. 行业与题材映射：必须支持历史时点，不能用今天的成分回测过去。
5. 财务与预期代理：利润、营收、现金流、ROE、业绩预告和公告时间。
6. 市场数据：主要指数、市场广度、融资融券、ETF份额等。
7. 全球风险数据：主要股指、美元、人民币、利率、波动率和商品。
8. 公告与事件：按真实公开时间入库，保留原文和解析结果。

### 5.2 数据源策略

- 生产环境优先选择具备商业授权、稳定接口和历史修订说明的数据服务商。
- Tushare Pro可用于原型和数据完整性评估；其日线接口说明包含更新时间、复权和停牌处理方式。
- AKShare可用于原型补充和交叉核验，但其部分接口依赖第三方网页，不能作为商业产品的唯一生产数据源。
- 交易所、巨潮资讯等官方披露用于公告与规则核验。
- 所有外部数据通过Provider适配层接入，避免策略代码绑定单一供应商。

### 5.3 数据质量门禁

每日任务必须检查：

- 交易日是否齐全；
- 股票数量和成交额是否异常；
- 复权因子是否突变；
- 停牌、涨跌停与退市状态是否一致；
- 财务数据是否存在发布日期晚于特征日期；
- 题材成分是否发生历史穿越；
- 数据缺失时停止生成新建议，而不是静默使用旧数据。

## 6. 算法设计

系统采用分层、可解释模型。第一版以规则、横截面排名和简单统计模型为主，不直接上深度学习。

### 6.1 市场状态层

```text
MarketScore =
  25% A股趋势与市场广度
+ 20% 大小盘和成长价值风格
+ 15% 成交、融资和流动性
+ 15% 全球风险环境
+ 15% 涨跌停、创新高低等情绪
+ 10% 估值与风险溢价
```

输出总仓位上限：

| 市场状态 | 总仓位上限 |
|---|---:|
| 强势 | 80%～95% |
| 震荡偏强 | 60%～80% |
| 震荡偏弱 | 30%～50% |
| 下跌 | 0%～20% |
| 极端风险 | 0% |

### 6.2 题材层

```text
ThemeScore =
  30% 20/60日相对强度
+ 20% 题材内部上涨广度
+ 15% 成交额占比及变化
+ 15% 基本面或盈利改善
+ 10% 政策、公告与新闻催化
+ 10% 龙头稳定性
- 拥挤度和加速惩罚
```

每周选择3～4个题材。进入组合的题材应处于启动、扩散或健康趋势阶段；加速末端和退潮阶段原则上不新开仓。

### 6.3 个股层

先执行硬过滤：

- ST、退市风险和严重监管风险；
- 上市不足120个交易日；
- 长期停牌或难以成交；
- 流动性不满足账户容量；
- 财务和审计异常；
- 处于不可接受的重大事件窗口。

再计算：

```text
StockScore =
  25% 60日相对强度
+ 20% 盈利质量和改善趋势
+ 15% 量价与成交结构
+ 15% 题材核心度
+ 10% 中期趋势质量
+ 10% 流动性和容量
+  5% 催化剂
- 波动、拥挤和尾部风险惩罚
```

只有同时满足市场允许、题材分数达标、个股分数达标、流动性达标和组合相关性约束的股票才能进入正式组合。没有足够合格股票时允许持有现金，不强行凑满5只。

### 6.4 3～5只精选组合构建

- 默认4只，允许3～5只；
- 至少覆盖2个相对独立题材；
- 同一题材最多2只；
- 同一产业链合计不超过45%；
- 单只股票目标仓位15%～25%，硬上限25%；
- 3只持仓时保留至少25%现金；
- 单股计划仓位原则上不超过其20日日均成交额的1%～2%；
- 使用相关性惩罚，避免名称不同但风险来源相同的伪分散。

仓位按风险计算：

```text
目标仓位 = 单笔风险预算 / 初始止损距离
```

单笔风险预算初始设为净值的1.0%～1.5%，再受单股仓位上限、市场总仓位和流动性约束。

### 6.5 买入规则

支持两类信号：

1. 平台突破：市场允许、题材健康、个股高分、突破40～60日平台且成交结构确认。
2. 趋势回踩：中期趋势向上、回踩20/30日趋势区间、缩量企稳并重新转强。

首次只建立计划仓位的50%～70%，后续确认趋势再加至目标仓位。回测统一采用信号次日可成交价格，禁止使用当日收盘后才能获得的信息按当日收盘成交。

### 6.6 卖出规则

退出优先级：

1. 财务、退市、处罚等硬风险；
2. 组合级回撤风控；
3. 市场进入极端风险状态；
4. 题材连续衰减或进入退潮；
5. 个股趋势和相对强度失效；
6. ATR初始止损或移动止盈；
7. 持有40～80日后仍无超额收益的时间退出；
8. 出现显著更优候选时的低频替换。

浮盈保护公式：

```text
保护价 = 买入价 + 70% ×（持仓最高价 - 买入价）
```

仅当浮盈达到15%左右后启动。在此之前使用基于ATR和结构低点的初始止损，避免小幅盈利时被普通波动反复洗出。

## 7. 回测与研究规范

### 7.1 必须消除的偏差

- 幸存者偏差；
- 使用当前行业/题材成分回测历史；
- 财务数据发布日期穿越；
- 用修订后的宏观数据替代当时可见数据；
- 停牌、涨跌停时假设可以成交；
- 忽略退市股票和公司行为；
- 用收盘后信号按当日收盘成交；
- 反复调参后只展示最优结果。

### 7.2 成交模拟

至少纳入：

- 佣金、印花税和其他必要费用；
- 买卖价差与滑点；
- 100股等交易单位和股数取整；
- 涨跌停、停牌和成交量约束；
- 大额账户分批成交；
- 信号生成、下单和成交的时间顺序。

### 7.3 验证方法

- 研究区间尽量覆盖2015年以来多轮牛熊和风格切换；
- 使用滚动训练/验证/测试，不一次性看到全部未来；
- 保留最终完全未参与调参的样本外区间；
- 与沪深300、中证全指及简单动量基线比较；
- 分资金档位模拟10万、100万、300万和1000万元的容量；
- 分析每个因子的边际贡献，不能只看组合最终收益。

### 7.4 MVP验收指标

以下是研究门槛，不是收益承诺：

- 样本外最大回撤目标不高于15%，硬门槛18%；
- 平均持仓数位于3～5只；
- 持仓中位数位于40～80个交易日；
- 策略在扣除费用后具有正的样本外超额收益；
- 相比无风控版本，市场状态层能实质降低回撤；
- 结果不依赖单一年份、单一题材或极少数股票；
- 参数在邻近区间内稳定，而不是只有一个参数点有效；
- 1000万元模拟账户不存在明显不可成交问题。

若不能达到门槛，应返回研究阶段调整假设，而不是上线包装。

## 8. 技术架构

建议采用前后端分离的单体架构起步，暂不拆微服务。

```text
外部数据源
   ↓
数据采集与质量检查
   ↓
原始数据层 → 清洗/时点化数据层 → 特征层
                                  ↓
市场状态 → 题材评分 → 个股评分 → 组合与风控
                                  ↓
回测/模拟盘 ← 决策快照与审计日志 → API
                                  ↓
                              Web前端
```

0.4.0 当前实现技术栈：

- 研究与服务端：Python、FastAPI；
- 数据处理：Python 标准库与 Provider 适配层；后续可按真实数据规模引入 Polars/Pandas；
- 研究制品：哈希锁定的 JSON 证据包；
- 产品数据库：SQLite（WAL、在线备份）；规模扩大时再评估 PostgreSQL；
- 任务调度：systemd timers；规模扩大后再评估队列；
- 前端：React＋Vite＋TypeScript；
- 图表：支持K线、组合净值、题材生命周期和风险分解的图表库；
- 测试：单元测试、数据快照测试、回测回归测试；
- 部署：Docker Compose起步，保留迁移云环境的能力。

原则：研究代码与生产信号必须共享同一套特征和规则实现，防止“回测一套、上线另一套”。

工程状态：市场/题材/个股/组合/风控、T+1 成交约束、持仓状态和研究制品已实现；生产建议与回测调用共享持仓策略。真实绩效结论仍取决于阶段3的授权数据研究。

### 阶段3：稳健性验证（FAIL / 等待真实数据证据）

- 滚动样本外测试；
- 参数敏感性、因子消融和压力测试；
- 10万～1000万元容量测试；
- 检查极端行情、停牌、跌停和数据缺失场景；
- 冻结MVP模型版本。

当前结论：研究管线能够生成门禁、基线、容量、敏感性、消融、压力与归因制品，但没有授权 PIT 数据和 2015 年以来最终隔离样本外 PASS 结果；必须保持发布门禁 FAIL。

### 阶段4：产品化（工程候选）

- 实现组合首页、市场温度、题材雷达、个股详情和决策日志；
- 提供每日收盘后任务和异常告警；
- 上线模拟组合，不连接真实交易；
- 建立模型版本、数据版本和推荐记录审计。

工程状态：页面、API、日终、决策日志与输入快照回放、研究工作台、备份/恢复和 CI 已具备候选能力；当前工作树仍须通过最终整仓回归和部署验收，不能称为正式 MVP。

### 阶段5：模拟运行（FAIL / 尚未完成时间门槛）

- 每日固定时间生成建议；
- 记录实际可成交价格和偏差；
- 比较回测、模拟成交与真实市场表现；
- 只修复工程问题，不根据短期盈亏频繁改策略；
- 模拟期结束后决定继续研究、开放内测或终止。

## 11. 测试与上线门禁

### 工程测试

- 核心因子和仓位计算单元测试；
- 除权、停牌、退市、涨跌停等边界测试；
- 历史数据重放测试；
- 每日信号结果回归测试；
- 数据源中断和重复写入测试；
- 权限、密钥、日志脱敏和备份恢复测试。

### 产品上线门禁

- 数据许可允许目标使用方式；
- 策略样本外结果达到预设门槛；
- 模拟运行不少于8～12周；
- 所有建议具有原因和失效条件；
- 风险提示清晰，不展示保证性语言；
- 不存在自动或半自动下单、券商连接与真实订单能力；
- 若未来另立真实交易项目，先完成独立的程序化交易合规、安全与券商接入评估。

## 12. 主要风险与应对

| 风险 | 应对 |
|---|---|
| 只有3～5只导致个股风险过高 | 单股上限、题材约束、事件过滤、现金仓位 |
| 题材分类存在未来信息 | 保存历史成分和每次变更生效日期 |
| 新闻热度诱导追高 | 新闻只做催化和风险解释，不直接决定买入 |
| “主力净流入”口径不可靠 | 使用成交结构、融资、ETF等可验证代理变量 |
| 免费数据源不稳定 | Provider抽象、质量门禁、生产数据商业授权 |
| 回测过拟合 | 样本外测试、参数稳定性、基线对照、因子消融 |
| 1000万元低流动性冲击 | 成交额比例约束、分批建仓、容量分档 |
| 自动交易合规和技术风险 | MVP不自动下单，后续独立立项 |
| 用户把建议当作收益承诺 | 风险揭示、可解释性、禁止保证性表述 |

## 13. 正式发布前必须冻结的决定

1. 股票范围：当前默认主板、创业板和科创板，排除北交所；正式模型冻结时确认是否保持该范围。
2. 数据预算与供应商：公开 AKShare 仅用于原型/观察；正式发布必须取得覆盖所需历史的可商用、可追溯 PIT 数据及书面授权。
3. 产品形态：个人自用工具，还是未来向外部用户提供服务？
4. 组合最大回撤：是否接受暂定的目标15%、硬风控18%？

这些决定不阻塞工程候选继续验证，但在确认前不得冻结正式策略参数，也不得把当前公开源观察版包装为生产系统。
