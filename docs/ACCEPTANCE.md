# MVP 验收矩阵

## 1. 验收规则

验收不是“页面能打开”。每项必须提供可复现证据：测试命令及完整结果、固定数据快照/模型版本、API 响应或截图、报告文件和日志引用。`REQUIREMENTS.md` 中任一强制需求无证据、证据与范围不匹配或测试只覆盖 happy path，均视为未完成。

测试环境必须记录：Git commit、依赖锁文件、数据库迁移版本、数据 snapshot、模型/特征版本、时区、随机种子和运行命令。Demo 证据只能证明功能，不可证明真实数据有效性或策略收益。

## 3. 必须具备的自动化测试层

1. **领域单元测试**：评分、仓位、约束、保护价、退出优先级。
2. **属性/不变量测试**：权重不超上限、保护价不下降、现金与持仓守恒、未来数据不可见。
3. **Provider 契约与数据质量测试**：真实 provider 可用受环境控制；CI 至少跑确定性 fixture。
4. **API 集成与 schema 测试**：数据库迁移、错误结构、分页、幂等和权限。
5. **回测 golden regression**：固定小数据集的信号、成交、净值和指标均有容差明确的金样。
6. **前端组件/状态测试**：所有关键状态。
7. **端到端测试**：设置资金→查看日终建议→展开解释→应用模拟→查看审计；回测启动→完成/失败→比较。
8. **无障碍与响应式测试**：自动 axe 加人工键盘/读屏抽测。
9. **安全与恢复测试**：依赖/密钥扫描、鉴权、日志脱敏、备份恢复。

## 4. 研究门禁证据包

每个候选模型版本必须产出不可变目录或制品，至少包括：

- `manifest.json`：commit、模型/特征/数据版本、配置 hash、随机种子；
- 数据覆盖与质量报告；
- 样本切分及“最终测试集未参与调参”的声明和运行记录；
- 全参数实验索引、敏感性图、因子消融、压力测试；
- 四资金档成交容量及成本报告；
- 三类基线对比；
- 逐年、题材、股票贡献和极端事件结果；
- `gates.json`：BT-008 每项机器可读 PASS/FAIL；
- 完整成交台账和可复现命令。

样本外最大回撤超过 18%、扣费后超额不为正、持仓/持有期不满足，或结果集中于单一年份/题材/极少股票时，研究门禁必须 FAIL。目标 15% 是优化目标，18% 是不可越过的上线硬门槛，均不是收益或风险保证。

## 5. 发布门禁

全部满足后方可称为“功能开发完成”：

- 自动化测试全绿且无跳过核心测试；
- OpenAPI 与前端契约一致，生产构建成功；
- 所有关键 UI 状态、响应式和 WCAG AA 验证通过；
- Demo 与真实数据明确区分，生产数据授权已书面确认；
- 研究门禁 PASS；
- 决策回放、日终幂等、数据过期抑制和备份恢复通过；
- 安全扫描无高危未处理项；
- 风险提示与“无自动交易”边界通过人工审查；
- 模拟盘已经真实连续运行至少 8～12 周。

最后一项是时间型门禁：代码完成不能替代真实观察期。未满 8 周时，系统最多标记为“工程候选版/模拟观察中”，不能宣称 MVP 全部验收完成。
