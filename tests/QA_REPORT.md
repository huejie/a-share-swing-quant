# 0.4.0 工程候选验收报告

日期：2026-07-16
模型基线：`swing-rules-0.4.0`
总体结论：**工程候选 PARTIAL；正式 MVP / 生产发布门禁 FAIL**
产品边界：研究、回测、每日建议和模拟组合；**自动交易明确排除**

## 1. 结论与证据边界

当前仓库已具备前后端、Provider 适配、策略规则、回测/研究框架、模拟账务、决策审计、日终任务和 Linux 运维候选能力。但本报告不把“代码存在”“Demo 可运行”或“公开接口能取数”解释为策略有效或正式发布通过。

截至本报告更新时，0.4.0 候选工作树已完成 187 项 Python 测试、39 项前端单元测试、生产构建、7 项真实 Chromium E2E、`npm audit` 与 `pip-audit`；本地检查均通过。服务器同提交部署、GitHub Actions 和周期恢复演练仍需在提交后核验。

正式发布仍缺三类不可替代的外部证据：

1. 目标用途书面授权的完整 PIT 数据包；公开 AKShare 数据是非 PIT、非生产授权的观察数据。
2. 使用该授权包完成的 2015 年以来研究，以及冻结参数后的最终隔离样本外门禁 PASS。
3. 冻结模型与数据口径后真实、连续 8～12 周的前瞻模拟观察。

因此，本版本只能称为“工程候选版 / 模拟观察准备中”，不能宣称正式 MVP 通过、策略有效或可投入生产决策。
