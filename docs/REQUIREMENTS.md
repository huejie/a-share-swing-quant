# A股中期波段精选系统需求规格

版本：1.0  
基线来源：`PROJECT_PLAN.md`、`.impeccable.md`  
适用范围：MVP（研究、回测、每日建议、模拟组合）；明确排除自动交易

## 1. 产品目标与术语

系统面向管理人民币 10 万～1000 万元的个人 A 股投资者，在每个交易日收盘后，生成可解释、可追溯、可模拟执行的 3～5 只精选组合建议。目标持有周期约 3 个月，即 40～80 个交易日；周频选择题材与常规调仓，日频检查风险。

- **模型建议**：策略根据时点数据产生的目标动作，不代表真实成交。
- **模拟持仓**：模拟成交引擎记录的虚拟持仓，与用户真实账户严格区分。
- **正式持仓**：模型组合中的有效标的；备选股不计入。
- **保护价**：浮盈达到启动门槛后，按最高浮盈回吐 30% 计算的移动止盈价。
- **数据时点**：该决策可见数据的最晚公开时间；不得以采集时间替代公开时间。

## 2. 强制产品边界

| ID | 可验证需求 |
|---|---|
| BND-001 | 系统只提供研究、决策支持、回测和模拟组合，不连接券商，不提交、撤销或管理真实订单。 |
| BND-002 | 代码、配置、API 和 UI 均不得包含“自动交易”“一键买入”等可触发真实交易的能力；不得保存券商交易凭据。 |
| BND-003 | 所有建议页持续展示“研究与决策辅助，不构成收益保证，不会自动执行”的风险提示。 |
| BND-004 | UI 必须以明确标签区分“模型输出”“模拟持仓”“用户自行记录的真实持仓”（若未来提供记录能力）；三者不得合并计算或混称。 |
| BND-005 | MVP 股票范围默认包含主板、创业板、科创板，排除北交所、B 股、基金、债券及衍生品；范围可配置且写入决策快照。 |

## 3. 用户与组合需求

| ID | 可验证需求 |
|---|---|
| PRT-001 | 支持账户规模 10 万、100 万、300 万、1000 万元及范围内自定义金额，并在仓位计算中应用 100 股交易单位与流动性约束。 |
| PRT-002 | 正常建议持有 3～5 只，默认目标 4 只；备选池 2～3 只且不计入持仓。 |
| PRT-003 | 合格股票不足 3 只时允许 0～2 只或空仓，并明确显示“未凑满”和现金比例；禁止用低分股票补足数量。 |
| PRT-004 | 至少覆盖 2 个相对独立题材；同题材最多 2 只，同产业链目标权重合计不超过 45%。不足条件时保留现金。 |
| PRT-005 | 单股目标权重 15%～25%，硬上限 25%；持有 3 只时现金至少 25%。所有权重之和不得超过市场状态给出的总仓位上限。 |
| PRT-006 | 单笔风险预算默认净值 1.0%～1.5%；原始仓位按“风险预算÷初始止损距离”计算，再受单股、总仓位、流动性和取整限制。 |
| PRT-007 | 单股计划金额原则上不得超过其 20 日日均成交额的 1%～2%，比例须配置并进入模型版本。 |
| PRT-008 | 策略目标持有 40～80 个交易日；时间退出不覆盖硬风险、市场风险、题材退潮、趋势失效和止损等优先退出条件。 |
| PRT-009 | 正常情况下每周最多替换 1 只；硬风险和组合风控触发的退出不计作常规替换。 |

## 4. 数据与时点需求

| ID | 可验证需求 |
|---|---|
| DAT-001 | Provider 适配层覆盖日线 OHLCV/成交额、复权与公司行为、证券状态、上市退市、板块、ST、流通市值。 |
| DAT-002 | 覆盖历史时点化行业/题材映射、财务及公告公开时间、主要指数、市场广度、融资融券、ETF 份额等可得数据。 |
| DAT-003 | 全球风险至少可表达全球股指、美元/人民币、利率、波动率和商品；不可得分项必须显式降级，不得伪造。 |
| DAT-004 | 公告与事件保存原文引用、实际公开时间、采集时间和解析版本；特征只能在公开时间之后生效。 |
| DAT-005 | 原始层不可变；清洗层、特征层和决策层记录 provider、数据批次、as-of 时间、生成时间及 schema 版本。 |
| DAT-006 | 指定历史交易日时，能够重建当日可见证券池、题材映射、特征、策略配置和建议。 |
| DAT-007 | 日终门禁检查交易日完整性、证券数量/成交额异常、复权突变、停牌/涨跌停/退市一致性、发布日期穿越、题材穿越和关键字段缺失。 |
| DAT-008 | 关键数据缺失、过期或质量门禁失败时，不生成新的“健康”建议；保留上一份快照但醒目标注过期，不静默沿用。 |
| DAT-009 | Demo provider 必须确定性、可离线复现并明确标识“演示数据”；不得在 UI 中与生产/授权数据混淆。 |

## 5. 策略需求

### 5.1 市场状态

| ID | 可验证需求 |
|---|---|
| MKT-001 | MarketScore 包含 A 股趋势与广度 25%、风格 20%、成交/融资/流动性 15%、全球风险 15%、情绪 15%、估值与风险溢价 10%；缺失处理须显式记录。 |
| MKT-002 | 输出强势、震荡偏强、震荡偏弱、下跌、极端风险之一，并给出分项分数、主要原因、置信/完整性和数据时间。 |
| MKT-003 | 总仓位上限分别落入 80%～95%、60%～80%、30%～50%、0%～20%、0%；配置变化须版本化。 |
| MKT-004 | 市场状态不得直接决定具体股票；它只约束总仓位、信号资格和风险动作。 |

### 5.2 题材与资金

| ID | 可验证需求 |
|---|---|
| THM-001 | ThemeScore 表达 20/60 日相对强度 30%、内部广度 20%、成交额占比变化 15%、基本面改善 15%、催化 10%、龙头稳定性 10%，另扣拥挤/加速惩罚。 |
| THM-002 | 每周排序并识别潜伏、启动、扩散、健康趋势、加速、拥挤、退潮；新开仓只允许启动、扩散或健康趋势。 |
| THM-003 | 正常选择 3～4 个题材；每个题材显示强度、广度、成交占比、催化持续性、拥挤度及生命周期依据。 |
| THM-004 | “资金流向”使用可验证代理变量（成交结构、融资、ETF 等）并标注口径；不可将推算的“主力净流入”包装为真实账户流向或核心事实。 |
| THM-005 | 新闻/大模型输出只能解释催化和风险，不得单独触发买入，不得作为未来涨跌标签。 |

### 5.3 个股选择与信号

| ID | 可验证需求 |
|---|---|
| STK-001 | 入池前排除 ST/退市和严重监管风险、上市不足 120 个交易日、长期停牌/难成交、容量不足、财务审计异常及不可接受事件窗口。 |
| STK-002 | StockScore 表达 60 日相对强度 25%、盈利质量改善 20%、量价结构 15%、题材核心度 15%、趋势质量 10%、流动性容量 10%、催化 5%，并扣波动/拥挤/尾部风险。 |
| STK-003 | 正式组合须同时通过市场、题材、个股、流动性、相关性和风险门槛；结果记录每一门槛的通过/失败原因。 |
| SIG-001 | 平台突破信号要求市场允许、题材健康、个股高分、突破 40～60 日平台并获成交结构确认。 |
| SIG-002 | 趋势回踩信号要求中期趋势向上、回踩 20/30 日趋势区间、缩量企稳并重新转强。 |
| SIG-003 | 首次建议仓位为计划仓位的 50%～70%，只有后续确认信号才能加至目标。 |
| SIG-004 | 每条建议包含动作、目标权重、入场状态/区间、最多 3 条入选理由、失效条件、初始止损、保护价、风险事件、预计持有期、下次复核时间、模型版本和数据时间。 |
| SIG-005 | 退出按硬风险、组合风控、极端市场、题材退潮、个股失效、ATR/移动止盈、时间退出、低频替换的优先级执行并记录触发源。 |
| SIG-006 | 浮盈约达 15% 后启动保护价：`entry + 70% × (peak - entry)`，即最多回吐最高浮盈的 30%；此前使用 ATR/结构止损。 |
| SIG-007 | 保护价只随新的持仓最高价上移，不得下调；除权等公司行为必须对 entry、peak 和保护价作一致调整。 |

## 6. 回测、模拟盘与审计

| ID | 可验证需求 |
|---|---|
| BT-001 | 回测与生产建议调用同一套特征、策略、组合和风控实现；配置与模型版本一致可追踪。 |
| BT-002 | 信号在收盘后生成，最早按次日可成交价格执行；禁止以当日收盘价成交收盘后信号。 |
| BT-003 | 回测处理退市/历史证券、历史题材成分、公告发布日期、当时可见宏观数据、复权/公司行为、停牌、涨跌停及不可成交。 |
| BT-004 | 成交模型处理佣金、印花税等费用、价差/滑点、100 股取整、成交量限制和大账户分批成交。 |
| BT-005 | 支持 2015 年以来（数据可得范围内）的滚动训练/验证/测试、最终隔离样本外、参数敏感性、因子消融和压力测试。 |
| BT-006 | 对比沪深 300、中证全指及简单动量基线；报告净值、超额、回撤、换手、胜率、盈亏比、持有期、容量和分年度/题材/股票贡献。 |
| BT-007 | 分别模拟 10 万、100 万、300 万、1000 万元，验证流动性和可成交性。 |
| BT-008 | 研究上线门槛：样本外最大回撤目标≤15%、硬门槛≤18%；平均持仓 3～5；持仓中位数 40～80 日；扣费后样本外超额为正；结果不依赖单一年份/题材/极少股票且邻近参数稳定。未达门槛必须显示“不通过”，不得包装上线。 |
| SIM-001 | 模拟盘按当时真实可见信号和下一可成交时点记录委托意图、模拟成交、未成交、部分成交、费用、滑点和持仓。 |
| SIM-002 | 模拟盘支持 0～5 只持仓、现金、净值、组合回撤、市场/题材/单股风险动作和 2～3 只备选股。 |
| SIM-003 | 模拟期要求至少连续 8～12 周；工程修复与策略变更分开记录，策略变更必须新建模型版本，禁止覆盖历史。 |
| AUD-001 | 每次日终运行保存输入数据批次、质量结果、特征摘要、模型/配置版本、候选排序、过滤原因、最终建议和模拟结果。 |
| AUD-002 | 任一历史建议可回放“当时为何这样判断”，审计记录追加写且不可通过普通产品 API 修改或删除。 |

## 8. 非功能与安全需求

| ID | 可验证需求 |
|---|---|
| NFR-001 | 相同数据快照、配置、随机种子和模型版本产生字节级或数值容差内一致结果。 |
| NFR-002 | API 采用版本前缀；时间使用带时区 ISO 8601，交易日使用 `YYYY-MM-DD`，价格/金额/权重明确单位和精度。 |
| NFR-003 | 密钥只来自环境或密钥服务，不入库、不进前端包、不写日志；日志对 token、个人设置等敏感字段脱敏。 |
| NFR-004 | 日终任务可重复执行且幂等；相同 run key 不得重复产生模拟成交。 |
| NFR-005 | 关键数据库每日备份，并通过定期恢复演练证明可恢复；审计快照有独立保留策略。 |
| NFR-006 | 健康接口区分服务存活、数据库、provider、新鲜度与最近成功日终任务，不以 HTTP 200 掩盖降级。 |

## 9. 明确不在 MVP 范围

- 券商连接、真实订单、自动或半自动下单、撤单、资金划转；
- 分钟级/日内高频、打板和自动追涨停；
- 收益承诺、个性化代客理财或替用户承担投资决定；
- 以大语言模型直接预测涨跌；
- 未经数据授权的商业发布；
- 北交所、港股、美股和衍生品交易策略。
